# SynoHub Project Guide For ChatGPT

This zip contains the SynoHub React + Express + MySQL project.

## What The App Does

SynoHub is a fleet CRM and service-ticket assistant for Synosys Fleet Intelligence.
It manages:

- Customers and company records
- Lead registrations
- Service requests and tickets
- Technician/staff assignment views
- Guest/staff/admin access control
- AI chat assistance for CRM operations

## Main Runtime Files

- `server.ts` - Express API server, auth, DB endpoints, AI routing, record save logic.
- `src/App.tsx` - Main React frontend UI and chat interface.
- `src/components/LoginForm.tsx` - Login form.
- `src/db/schema.ts` - Drizzle table definitions for MySQL.
- `src/db/index.ts` - MySQL/Drizzle connection setup.
- `queryRegistry.ts` - Safe backend query registry. This prevents AI-generated SQL.
- `prompts.json` - Main AI assistant prompt text.
- `src/ai/intentDetector.py` - Local Python intent detector for safe query classification.
- `src/ai/local-llm/systemPrompt.txt` - Local LLM behavior prompt.

## Database Files

- `database_export/synohub_live_project_tables.sql` - Live export of the project tables only.
- `synohub_full_sample_schema_and_data.sql` - Sample full schema and sample data.
- `service_requests_1000.sql` - Sample service request data.

The important tables are:

- `customers`
- `service_requests`
- `messages`
- `salesplus_entries`

## AI System Overview

The app has two AI routes:

1. `/api/chat`
   General conversational assistant. It can draft registrations/service tickets and can output internal save triggers that the server processes.

2. `/api/chat/query`
   Safe natural-language query route. AI only classifies intent. SQL is never generated by AI. The backend maps intents to fixed SQL in `queryRegistry.ts`.

Supported AI modes:

- Gemini
- Local LLM via Ollama
- GPT OSS 120B via OpenRouter
- Cohere via OpenRouter
- Compare mode

Chat history is stored in `messages.username` and scoped by user and model, for example:

- `guest:guest-id|ai:local`
- `guest:guest-id|ai:gemini`
- `guest:guest-id|ai:nvidia`
- `guest:guest-id|ai:openrouter`
- `guest:guest-id|ai:compare`

## Permission Model

- Admin can see all records.
- Staff can see records where `requested_person`, `sales_person`, or `created_by` matches their name.
- Guest can see only records where `created_by` matches their guest account.
